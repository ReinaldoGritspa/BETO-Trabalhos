Tag / Atributos Descrição Exemplo prático
<form>

Atributos:
action — URL de destino dos dados
method — GET | POST
enctype — codificação (ex: multipart/form-data para arquivos)
target — _self | _blank | _parent | _top
novalidate — desativa validação nativa
autocomplete — on | off 		Contêiner principal que agrupa os campos de um formulário.
<input type="text">

Atributos:
name — chave enviada ao servidor
value — valor inicial
placeholder — texto de dica
maxlength / minlength
required — campo obrigatório
readonly — somente leitura
disabled — desativado
autofocus — foco automático
autocomplete — on | off
pattern — regex de validação
size — largura visual em caracteres 		Campo de texto de linha única.
<input type="password">

Atributos: mesmos do type="text"		 Campo de senha — caracteres ocultados.
<input type="email">

Atributos: mesmos do type="text" +
multiple — aceita vários e-mails separados por vírgula		 Campo de e-mail com validação nativa de formato.
<input type="number">

Atributos:
min — valor mínimo
max — valor máximo
step — incremento (ex: 0.5)		 Campo numérico com controles de incremento/decremento.
<input type="range">

Atributos: min, max, step, value		 Controle deslizante (slider) para valores numéricos.
<input type="date">

Atributos: min, max, value		 Seletor de data (ano-mês-dia).
<input type="time">

Atributos: min, max, step		 Seletor de horário (horas e minutos).
<input type="datetime-local">

Atributos: min, max		 Seletor de data e hora combinados (sem fuso horário).
<input type="checkbox">

Atributos:
name — checkboxes com mesmo name formam um grupo
value — valor enviado quando marcado
checked — marcado por padrão
required		 Caixa de seleção. Múltiplas podem ser marcadas ao mesmo tempo.
Particularidade: se não marcado, o campo não é enviado ao servidor.
<input type="radio">

Atributos:
name — agrupa botões (mesmo name = mesmo grupo)
value — valor enviado quando selecionado
checked — selecionado por padrão		 Seleção exclusiva dentro de um grupo.
Particularidade: todos com o mesmo name são mutuamente exclusivos.
<input type="file">

Atributos:
accept — tipos aceitos (ex: .pdf, image/*, .jpg,.png)
multiple — permite selecionar vários arquivos		 Seletor de arquivo.
Particularidade: o formulário deve ter enctype="multipart/form-data".
<input type="color">

Atributos: value — cor padrão em hexadecimal		 Seletor de cor.
<input type="hidden">

Atributos: name, value		 Campo invisível. Envia dados sem exibir ao usuário (ex: token CSRF, ID de sessão).
<input type="search">

Atributos: mesmos do type="text"		 Campo de busca. Pode exibir botão de limpar (×) dependendo do navegador.
<input type="tel">

Atributos: pattern, placeholder, mesmos do type="text"		 Campo de telefone. Em dispositivos móveis abre teclado numérico.
<input type="url">

Atributos: mesmos do type="text"		 Campo de URL com validação nativa de formato.
<input type="submit">

Atributos:
value — texto do botão
formaction — sobrescreve action do form
formmethod — sobrescreve method do form
formnovalidate — ignora validação		 Botão que envia o formulário.
<input type="reset">

Atributos: value — texto do botão		 Botão que restaura todos os campos ao valor inicial.
<input type="button">

Atributos: value — texto do botão		 Botão sem comportamento padrão. Ação definida via JavaScript.
<button>

Atributos:
type — submit (padrão) | reset | button
name, value
disabled
formaction, formmethod, formnovalidate

Particularidade: pode conter HTML interno (ícones, imagens, texto formatado). 		Botão mais flexível que <input type="button">.
<textarea>

Atributos:
rows — número de linhas visíveis
cols — número de colunas visíveis
maxlength / minlength
placeholder
readonly, disabled, required
wrap — soft | hard 		Área de texto multilinha.
Particularidade: o valor padrão fica entre as tags, não no atributo value.
<select>

Atributos:
name
multiple — permite selecionar várias opções
size — nº de opções visíveis simultaneamente
required, disabled

<option>
Atributos:
value — valor enviado ao servidor
selected — selecionado por padrão
disabled — opção desativada
label — rótulo alternativo

<optgroup>
Atributos:
label — título do grupo (obrigatório)
disabled — desativa o grupo inteiro 		Lista de seleção.
Particularidades:
— O texto entre <option> é o que o usuário vê; value é o que é enviado.
— Se value for omitido, o texto visível é enviado.
— <optgroup> agrupa opções com um rótulo visual (não é selecionável).
— Com multiple, use Ctrl/Cmd + clique para selecionar várias.
<datalist>

Atributos:
id — deve corresponder ao list do input

Contém <option> com value 		Fornece sugestões (autocomplete) para um <input>.
Particularidade: o usuário ainda pode digitar um valor fora da lista (diferente de select). 		Linguagem:
<label>

Atributos:
for — deve corresponder ao id do campo associado (ou envolver o campo diretamente, sem for) 		Rótulo associado a um campo.
Particularidade: clicar no label foca/ativa o campo correspondente — importante para acessibilidade.
<fieldset>

Atributos:
disabled — desativa todos os campos internos
name, form

<legend>
Sem atributos específicos. Texto descritivo do grupo. 		Agrupa campos relacionados visualmente com uma borda.
Particularidade: disabled no fieldset desativa todos os campos filhos de uma só vez.
Dados pessoais
<output>

Atributos:
for — IDs dos inputs que originam o valor
name, form 		Exibe o resultado de um cálculo ou ação (geralmente atualizado via JavaScript).
+ = 0
<progress>

Atributos:
value — valor atual
max — valor máximo (padrão: 1) 		Barra de progresso.
Particularidade: sem value exibe animação de progresso indeterminado. 		Determinado: 70%
Indeterminado:
<meter>

Atributos:
value — valor atual (obrigatório)
min / max — intervalo
low — limite inferior do "bom"
high — limite superior do "bom"
optimum — valor ideal 		Medidor escalar dentro de um intervalo conhecido.
Particularidade: diferente de <progress>, representa uma medida, não um progresso.
Muda de cor conforme low, high e optimum.
